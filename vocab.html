    • HTTP

    A standard set of rules used to transfer data over to your browser.

  • Client

    Any Workstation that requests information from a server.

  • Server

    A computer that manages resources and services to local computers when requested.

  • Web Browser

    a interfacr that takes you anywhere on internet. It takes information from other parts of the web and displays it on your computer or mobile device.

  • Website

    A website is contains multiple web pages under the same umbrella and is usually managed by one person sometimes multiple people.

  • Web page

    A simple document usually made with the HTML that can be accessed by any web browser.

  • JavaScript

    object oriented programming language that can be used to spice up your webpage and give user intecations

  • HTML

    Hypertext Markup Language are the basic building blocks of a webpage you can create one with it for the simplest webpage.

  • CSS

    Cascading style sheet lets you add fonts and color to your page and is i guess the interior designer.

  • URL

    Uniform Resource Locator is the unique identifier for finding specific websites it's like a finger print and matches to one website per URL.

  • IDE / text editor

    A computer program used for editing text related to programming and text files.

  • Command line / Terminal

    Is a user interface on the computer that lets you run programs navigate,create and delete, folders and files.

  • BASH

    Bourne Again Shell. Usually the main command processor used in Linux and Mac. It is a command line interpreter.

  • Operating system

    An operating system you can think of as a software that sits in between the user and the machine and joins them together giving the user a UI to use the computer.

  • Algorithm

    A series of instructions that are followed step by step by the computer to solve or find something.

  • Frontend vs Backend Development

    Frontend developers build with users in mind, designing user experience. Backend developers focus on the side of the website that does all the work when the user interacts with the front end.

  • Programming vs Coding

    Coding is writing code in a language understood by both machines and humans. Programming involves creating a structure for the programs and how it understand code.

  • Git vs Github

    Git is a Version control system that helps you manage and keep track of you code history. Github is a cloud based service that stores a copy of your code. You use git in order to check, upload or download code in Github.

    • Github terms:

  • push

    You are submitting changes from your local computer to Github to updating it.

  • pull

    You are checking for any changes in Github that are not in your local machine

  • fork

    When you want to copy someone elses repository from their Github profile in order for you to work on it without the user's main repository/folder.

  • clone

    When you want to make a copy of your repository from your Github to your local computer so that you can work on the code offline.

  • repository

    storage where your project is kept on Github
